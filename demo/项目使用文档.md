# Spring Boot Demo 项目使用文档

## 项目概述

本项目是一个基于Spring Boot 3.0的演示项目，展示了RESTful API开发、数据访问、测试、Docker部署等核心功能。

## 技术栈

- **框架：** Spring Boot 3.0
- **Java版本：** Java 17
- **构建工具：** Maven 3.5.0+
- **数据库：** 内存存储（可扩展为MySQL）
- **测试框架：** JUnit 5 + Spring Boot Test
- **容器化：** Docker + Docker Compose

## 安装和配置

### 1. 克隆项目

```bash
git clone <repository-url>
cd spring-boot-examples/demo
```

### 2. 导入IDE

**IntelliJ IDEA：**
1. File -> Open
2. 选择demo目录
3. 等待Maven依赖下载完成
4. 确认JDK版本设置为17

**Eclipse：**
1. File -> Import -> Existing Maven Projects
2. 选择demo目录
3. Finish完成导入

### 3. 配置文件

**application.properties（主配置）：**
```properties
# 应用名称
spring.application.name=demo

# 服务器端口
server.port=8080

# 日志配置
logging.level.root=INFO
logging.level.com.example.demo=DEBUG
```

**application-dev.properties（开发环境）：**
```properties
# 开发环境配置
spring.profiles.active=dev

# 数据库配置（示例）
# spring.datasource.url=jdbc:mysql://localhost:3306/demo_db
# spring.datasource.username=root
# spring.datasource.password=root
```

**application-prod.properties（生产环境）：**
```properties
# 生产环境配置
spring.profiles.active=prod

# 性能优化
server.tomcat.max-threads=200
server.tomcat.min-spare-threads=10
```

## 运行方式

### 方式1：Maven命令

```bash
# 编译项目
mvn clean compile

# 运行应用
mvn spring-boot:run

# 打包项目
mvn clean package

# 跳过测试打包
mvn clean package -DskipTests

# 运行测试
mvn test
```

### 方式2：直接运行jar文件

```bash
# 编译打包
mvn clean package -DskipTests

# 运行jar文件
java -jar target/demo-0.0.1-SNAPSHOT.jar

# 指定端口运行
java -jar target/demo-0.0.1-SNAPSHOT.jar --server.port=8081

# 指定配置文件运行
java -jar target/demo-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

### 方式3：Windows启动脚本

```bash
# 启动应用
.\run.bat start

# 停止应用
.\run.bat stop

# 查看帮助
.\run.bat
```

**脚本功能：**
- 自动创建logs目录
- 在新窗口中启动应用
- 支持启动和停止操作
- 提供使用说明

### 方式4：Docker容器

```bash
# 构建Docker镜像
docker build -t spring-boot-demo:latest .

# 运行Docker容器
docker run -d -p 8080:8080 --name spring-boot-demo-container spring-boot-demo:latest

# 查看容器日志
docker logs spring-boot-demo-container

# 停止容器
docker stop spring-boot-demo-container

# 启动已停止的容器
docker start spring-boot-demo-container

# 删除容器
docker rm spring-boot-demo-container
```

## API文档

### 基础URL

```
基础URL: http://localhost:8080
API前缀: /api/users
```

### API端点

#### 1. 获取所有用户

**请求：**
```bash
GET /api/users
```

**示例：**
```bash
curl http://localhost:8080/api/users
```

**响应：**
```json
[
  {
    "id": 1,
    "username": "张三",
    "email": "zhangsan@example.com",
    "age": 25
  },
  {
    "id": 2,
    "username": "李四",
    "email": "lisi@example.com",
    "age": 30
  }
]
```

#### 2. 根据ID获取用户

**请求：**
```bash
GET /api/users/{id}
```

**示例：**
```bash
curl http://localhost:8080/api/users/1
```

**响应：**
```json
{
  "id": 1,
  "username": "张三",
  "email": "zhangsan@example.com",
  "age": 25
}
```

#### 3. 创建用户

**请求：**
```bash
POST /api/users
Content-Type: application/json
```

**示例：**
```bash
curl -X POST http://localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -d '{
    "username": "王五",
    "email": "wangwu@example.com",
    "age": 28
  }'
```

**响应：**
```json
"创建用户王五"
```

#### 4. 更新用户

**请求：**
```bash
PUT /api/users/{id}
Content-Type: application/json
```

**示例：**
```bash
curl -X PUT http://localhost:8080/api/users/1 \
  -H "Content-Type: application/json" \
  -d '{
    "username": "张三更新",
    "email": "zhangsan_new@example.com",
    "age": 26
  }'
```

**响应：**
```json
"更新用户ID为1的信息为张三更新"
```

#### 5. 删除用户

**请求：**
```bash
DELETE /api/users/{id}
```

**示例：**
```bash
curl -X DELETE http://localhost:8080/api/users/1
```

**响应：**
```json
"删除用户ID为1"
```

## Docker部署

### Dockerfile说明

```dockerfile
FROM openjdk:17-slim          # 使用OpenJDK 17精简版基础镜像

WORKDIR /app                    # 设置工作目录

COPY target/demo-0.0.1-SNAPSHOT.jar app.jar  # 复制jar文件

EXPOSE 8080                     # 暴露8080端口

ENTRYPOINT ["java", "-jar", "app.jar"]  # 设置启动命令
```

### 构建和运行

```bash
# 1. 构建Maven项目
mvn clean package -DskipTests

# 2. 构建Docker镜像
docker build -t spring-boot-demo:latest .

# 3. 运行Docker容器
docker run -d -p 8080:8080 --name spring-boot-demo-container spring-boot-demo:latest

# 4. 查看容器状态
docker ps

# 5. 查看应用日志
docker logs spring-boot-demo-container

# 6. 测试API
curl http://localhost:8080/api/users
```

### Docker Compose（可选）

如果需要多容器编排，可以创建docker-compose.yml：

```yaml
version: '3.8'
services:
  app:
    build: .
    ports:
      - "8080:8080"
    depends_on:
      - mysql
    environment:
      - SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/demo_db
      - SPRING_DATASOURCE_USERNAME=root
      - SPRING_DATASOURCE_PASSWORD=root
      
  mysql:
    image: mysql:8.0
    environment:
      - MYSQL_ROOT_PASSWORD=root
      - MYSQL_DATABASE=demo_db
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql

volumes:
  mysql-data:
```

运行：
```bash
docker-compose up -d
```

## 常见问题

### 1. 端口被占用

**问题：**
```
Port 8080 is already in use
```

**解决：**
```bash
# 方式1：修改端口
server.port=8081

# 方式2：查找并停止占用进程
netstat -ano | findstr :8080
taskkill /PID <进程ID> /F
```

### 2. Java版本不匹配

**问题：**
```
Unsupported class file major version 61
```

**解决：**
- 确保安装JDK 17+
- 设置JAVA_HOME环境变量
- 验证版本：`java -version`

### 3. Maven依赖下载慢

**问题：**
Maven依赖下载速度慢或失败

**解决：**
```xml
<!-- 在pom.xml中添加阿里云镜像 -->
<mirrors>
  <mirror>
    <id>aliyun</id>
    <mirrorOf>central</mirrorOf>
    <name>Aliyun Maven</name>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
</mirrors>
```

### 4. Docker连接失败

**问题：**
```
failed to connect to docker API
```

**解决：**
- 启动Docker Desktop
- 检查Docker服务状态
- 重启Docker Desktop

### 5. 应用启动失败

**问题：**
应用启动后立即退出

**解决：**
```bash
# 查看详细日志
java -jar target/demo-0.0.1-SNAPSHOT.jar --debug

# 检查配置文件
# 查看端口冲突
# 检查依赖冲突
```

## 扩展功能

### 数据库集成

如需集成真实数据库，添加依赖：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
    <scope>runtime</scope>
</dependency>
```

### 缓存集成

添加Redis缓存支持：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
```

### 安全集成

添加Spring Security：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>
```
