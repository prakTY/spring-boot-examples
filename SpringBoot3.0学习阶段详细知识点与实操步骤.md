#### 知识点1.1：Spring Boot 3.0主要改进

**知识点1.1.1：Java 17支持和要求**

**概念解释：**
Spring Boot 3.0要求Java 17作为最低版本，这意味着应用可以利用Java 17的新特性。

**Java 17新特性：**
- Records：不可变数据类的简洁语法
- Pattern Matching：更强大的模式匹配
- Sealed Classes：限制类的继承
- Text Blocks：多行字符串的简化语法
- Enhanced Switch：更强大的switch表达式
- Records：简化数据类定义

**影响说明：**
- 需要升级JDK版本到17或更高
- 可能需要调整现有代码以适应新特性
- 可以利用新特性简化代码

**代码示例：**
```java
// 使用Records定义数据类
public record User(Long id, String name, String email) {
    public User {
        if (name == null || name.isBlank()) {
            throw new IllegalArgumentException("Name cannot be blank");
        }
    }
}

// 使用Pattern Matching
public String describeUser(Object user) {
    return switch (user) {
        case User u -> "User: " + u.name();
        case String s -> "String: " + s;
        case null, default -> "Unknown";
    };
}

// 使用Text Blocks
public String getWelcomeMessage(String name) {
    return """
        Welcome, %s!
        
        Thank you for joining our platform.
        We hope you enjoy your experience.
        """.formatted(name);
}
```

**知识点1.1.2：AOT编译和GraalVM支持**

**概念解释：**
AOT（Ahead-of-Time）编译是Spring Boot 3.0的重要特性，它允许将Java应用编译为原生可执行文件，从而实现更快的启动时间和更低的内存占用。

**GraalVM简介：**
GraalVM是一个高性能的JDK，支持AOT编译，可以将Java应用编译为原生可执行文件。

**优势说明：**
- 启动时间：从几秒减少到毫秒级
- 内存占用：显著降低，适合微服务
- 性能：接近原生应用的性能
- 资源使用：更少的CPU和内存消耗

**Spring Native项目：**
Spring Native是Spring生态系统的子项目，专门支持GraalVM原生镜像编译。

**代码示例：**
```java
// Spring Boot 3.0应用不需要特殊代码即可支持AOT
@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

// 但需要添加Spring Native依赖
// pom.xml中添加：
<dependency>
    <groupId>org.springframework.experimental</groupId>
    <artifactId>spring-native</artifactId>
    <version>0.12.0</version>
</dependency>
```

**实操步骤1.1.3：添加Spring Native依赖**

```xml

                    <goals>
                        <goal>test-generate</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>
```

**知识点1.1.3：云原生应用支持**

**概念解释：**
Spring Boot 3.0增强了云原生应用支持，包括健康检查、指标监控、配置管理等，与Kubernetes等云原生环境无缝集成。

**云原生特性：**
- 健康检查：提供应用健康状态
- 指标监控：暴露应用运行指标
- 配置管理：支持外部配置中心
- 服务发现：与Kubernetes服务发现集成
- 容器优化：优化容器化部署

**Kubernetes集成：**
Spring Boot 3.0提供了对Kubernetes的更好支持，包括：
- ConfigMap和Secret配置
- Service和Ingress自动配置
- 健康检查集成
- 指标导出

**代码示例：**
```java
// 启用Actuator端点
// application.properties中添加：
management.endpoints.web.exposure.include=health,info,metrics
management.endpoint.health.show-details=always

// 自定义健康检查
@Component
public class CustomHealthIndicator implements HealthIndicator {
    @Override
    public Health health() {
        return Health.up()
            .withDetail("custom", "Custom health check passed")
            .build();
    }
}
```

**实操步骤1.1.5：启用Actuator端点**

```properties
# application.properties中启用Actuator
management.endpoints.web.exposure.include=health,info,metrics,env
management.endpoint.health.show-details=always
management.endpoint.health.show-components=always

# 启用Kubernetes支持
management.health.probes.enabled=true
management.endpoint.health.probes.add-additional-paths=/liveness, /readiness
```

**知识点1.1.4：性能优化和安全增强**

**性能优化：**
- 启动时间优化：通过AOT编译和延迟初始化
- 内存使用优化：减少不必要的Bean创建
- 数据库连接池优化：合理配置连接池大小
- 缓存优化：使用Spring Cache抽象

**安全增强：**
- 默认安全配置：更严格的默认安全设置
- 依赖安全检查：自动检查依赖的安全漏洞
- CSRF防护：增强的跨站请求伪造防护
- 密码加密：更安全的密码存储方式

**代码示例：**
```java
// 性能优化配置
@Configuration
public class PerformanceConfig {
    
    @Bean
    public ThreadPoolTaskExecutor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(5);
        executor.setMaxPoolSize(10);
        executor.setQueueCapacity(25);
        executor.setThreadNamePrefix("async-task-");
        executor.initialize();
        return executor;
    }
}

// 安全配置
@Configuration
public class SecurityConfig {
    
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```
